[云计算]HCIE-Cloud 问答题 - FusionCompute

[toc]

一、名词解释

1. 介绍一下什么是FusionCompute？

FusionCompute是华为的虚拟化软件，可以实现物理资源的虚拟化，并且提供对虚拟化资源管理和调度的能力。

2. FusionCompute由哪些组件组成？

由 VRM 和 CNA 组成

  • VRM：虚拟资源管理，通过Web UI提供给管理员进行监控/维护/操作等便捷管理功能，对集群内的计算/存储/网络资源进行高效管理，通过REST API与CNA通信。功能如下：

    • 管理集群内虚拟机的生命周期以及在计算节点上的分布和迁移（计算）
    • 管理集群内的块存储资源（存储）
    • 管理集群内的网络资源（网络）

  • CNA：计算节点代理，指安装了华为虚拟化操作系统的物理机，由UVP和VNA组成。

    • UVP：统一虚拟化平台，可以实现对物理资源的虚拟化，并对这些虚拟化资源进行管理和调度，同时提供了管理接口。
    • VNA：虚拟节点代理，实现VRM与CNA之间的通信。接受VRM下发的命令，同时收集CNA的状态信息并反馈给VRM。

3. 什么叫集群？

在FusionCompute中，集群指对CNA主机的逻辑划分，可以根据硬件配置类型或者是业务类型进行划分，统一管理，是一个逻辑上的定义。

4. 介绍一下什么是HA以及做HA有什么要求？

HA 高可用，当一台虚拟机发生故障时，系统自动在资源池中另一可用计算节点上重新启动虚拟机。（用户有感知，从检测触发HA到重新拉起需要一定时间）

条件：

  • 计算资源：目的节点上有充足的CPU和内存资源。
  • 网络资源：目的节点与故障节点同处同一集群，同一二层网络（DVS），保证重新启动后的IP和MAC地址等信息不变。
  • 存储资源：数据存储类型要是共享存储。

5. 内存复用

定义：内存复用指的是在服务器物理内存一定的情况下，采用内存复用技术对内存进行分时复用，实现虚拟机内存总和可大于物理机内存总和，提高虚拟机密度，同时一定程度上节约了经济开销。

三种内存复用技术：

  • 内存气泡：通过内存气泡技术，将内存使用率较低的虚拟机的内存释放给内存使用率较高的虚拟机使用，从而提升内存利用率。
  • 内存置换：通过内存置换技术，将内存中长期未访问的数据置换到存储中并修改映射，当需要用的时候再置换回来。
  • 内存共享，写时复制：多台虚拟机共享一段内存空间做只读操作；当需要写数据时，重新开辟一块新的内存空间并修改映射，进行数据写操作。

6. 分布式资源调度

分布式资源调度，指的是FusionCompute提供虚拟化能力的同时，也为系统提供了统一的资源动态调度能力，实现系统资源的有效利用，减少业务系统的能耗。

具体体现：

  • 负载均衡：对一个集群内的主机进行监控，并根据之前系统管理员设定的规则，使集群内各个主机的CPU、内存等资源负载处于相对均衡的状态。

  • 动态节能调度：配合负载均衡使用，仅在负载均衡功能启用后才开启。对一个集群内的主机进行监控，业务量大的时候自动上电主机并将业务压力分摊到各个主机上实现负载均衡；业务量小的时候，将虚拟机集中迁移到部分主机上，对其余主机进行下电操作。

7. DRS

DRS 动态资源调度，指的是采用智能负载均衡算法，并结合动态电源管理功能，通过周期性的检查同一集群内的各个主机的负载情况，在不同主机间迁移虚拟机，从而实现同一集群中不同主机之间的负载均衡，最大程度的降低了系统能耗。

8. DPM

DPM 动态电源管理，指的是根据业务负载情况，智能的将主机上下电。负载低的时候将虚拟机集中到部分主机上，并对其余主机进行下电操作；负载高的时候自动上电部分主机，并将业务虚拟机进行迁移，使得集群内主机负载相对均衡。

9. DRS规则有哪些？

聚集虚拟机：列出的主机必须在同一主机上运行，一个虚拟机只能被加入一个聚集虚拟机规则中。

互斥虚拟机：列出的主机必须在不同主机上运行，一个虚拟机只能被加入一个聚集虚拟机规则中。

虚拟机到主机：关联一个虚拟机组到主机组，并设置规则。

  • 必须在主机组上运行：虚拟机组中的虚拟机必须在主机组中的主机运行。
  • 禁止在主机组上运行：虚拟机组中的虚拟机禁止在主机组中的主机运行。
  • 应该在主机组上运行：虚拟机组中的虚拟机应该（但不是必须）在主机组中的主机运行。
  • 不应该在主机组上运行：虚拟机组中的虚拟机不应该（但是可以）在主机组中的主机运行。

10. 资源动态调整

资源动态调整，指的是 FusionCompute 支持进行性虚拟机资源的动态调整，用户可根据业务情况动态的调整资源的使用情况。包括如下：

  • 离线/在线添加vCPU数目
  • 离线/在线调整内存大小
  • 离线/在线添加或删除网卡
  • 离线/在线挂载虚拟磁盘

11. IMC

异构迁移模式，根据Intel厂商不同代的CPU做调整，是虚拟机热迁移的限制条件之一。

设置 IMC 策略时需满足以下条件：

  • 集群下主机的CPU功能集必须大于目标基准功能集。
  • 集群下运行或休眠的虚拟机CPU功能集要求必须低于目标基准功能集。
  • 若不满足以上条件，可将相关虚拟机关机或迁移出集群后重新进行设置。

12. QoS

QoS 服务质量，在资源紧张的情况下，保障关键业务的体验度，FusionCompute中有包括 CPU QoS、内存 QoS、存储 QoS、网络 QoS。

（保护下限，控制上限）

  • CPU QoS
    • CPU资源份额：资源紧张时，按照份额比例进行CPU资源分配。
    • CPU资源预留：最少需要预留的CPU资源量，保护下限。
    • CPU资源限额：最多可占用的CPU资源量，控制上限。
  • 内存 QoS
    • 内存资源份额：资源紧张时，按照份额比例进行内存资源分配。
    • 内存资源预留：最少需要预留的内存资源量，保护下限。
    • 内存资源限额：最多可占用的内存资源量，控制上限。
  • 磁盘 QoS
    • 最大读/写字节吞吐量
    • 最多每秒读/写请求量
  • 网络 QoS
    • 平均带宽：平均可使用的网络带宽吞吐量。
    • 峰值带宽：峰值的网络带宽吞吐量。
    • 突发带宽：允许在平均带宽基础上，产生的突发带宽吞吐量。

13. 介绍一下Tools

定义：虚拟机的驱动程序，提高虚拟机的IO能力，实现对虚拟机的监控和其他一些高级功能。

  • 提高虚拟机网络与磁盘的IO性能
  • 实现虚拟机的硬件监控功能
    • 虚拟机指定网卡的IP信息
    • 虚拟机内部各CPU、内存的利用率
    • 虚拟机各磁盘的分区及使用情况
  • 实现虚拟机的高级特性
    • 在线调整虚拟机vCPU规格
    • 创建虚拟机快照
    • 虚拟机蓝屏检测
    • 虚拟机与主机的时钟同步
    • 虚拟机的网卡高级特性，如：QoS
    • 自动升级虚拟机的驱动程序，如：Tools

14. 介绍一下存储资源、存储设备和数据存储

存储资源：指物理存储设备，包括：NAS、IP SAN、FC SAN、FusionStorage Block

存储设备：指存储资源中的管理单元，如：NAS共享目录、LUN、FusionStorage存储池、本地磁盘。（一个存储资源可有多个存储设备）

数据存储：指FusionCompute对存储设备上的存储单元进行统一封装。与主机关联后可进一步创建出多个虚拟磁盘（一个存储设备对应一个数据存储）

  • 数据存储表示系统中可管理、操作的存储逻辑单元；
  • 一个数据存储对应一个存储设备；
  • 数据存储承载了虚拟机的业务；
  • 数据存储和主机关联，为主机提供资源，数据存储可以关联到多个主机，一个主机可关联多个数据存储；

15. 什么是DVS？

图例：上行链路、DVS、端口组

DVS 分布式虚拟交换机：实现系统管理员通过VRM提供的Web UI对一台到多台CNA主机上的虚拟交换机的物理端口和虚拟端口进行配置/维护。

原理：VRM发送控制指令给CNA上的虚拟交换机，由虚拟交换机来实现对数据的转发。

16. 什么是上行链路？

上行链路：指DVS虚拟接口到CNA主机物理网卡的链路，DVS通过上行链路关联CNA主机。

17. 什么是端口组？

端口组：指DVS中具有相同配置属性的虚拟端口的集合，如：VLAN、IP和MAC绑定、DHCP隔离等，减少了重复配置的工作。

18. OVS和EVS有什么区别？

OVS：Open vSwitch是一款开源的虚拟交换机。

EVS：Elastic vSwitch是华为基于OVS开发的增强版虚拟交换机，增加了其IO性能。

二、热迁移

1. 热迁移

热迁移：分为虚拟机热迁移、存储热迁移。

虚拟机热迁移：将“运行中”的虚拟机从一台主机迁移到另一台主机，迁移过程业务不中断（用户无感知）。

存储热迁移：将“运行中”的虚拟机的虚拟磁盘从一个数据存储迁移到另一个数据存储中，迁移过程业务不中断（用户无感知）

FusionCompute中，可得知的热迁移有三种：更改主机（虚拟机热迁移）、更改数据存储（存储迁移）、更改主机+数据存储（完整迁移）

补充：更改数据存储对应的是存储迁移，开关机都可以，所以不存在热这一说法。

2. 如何实现热迁移的？

  1. 更改主机：

    • 针对内存：内存分片 + 迭代迁移
    • 针对存储：共享存储

  2. 更改数据存储：

    • 针对存储：MirrorIO + 迭代迁移

  3. 更改主机+数据存储：

    • 针对内存：内存分片 + 迭代迁移
    • 针对存储：MirrorIO + 迭代迁移

3. 解释一下虚拟机热迁移的定义、迁移过程？

虚拟机热迁移：将“运行中”的虚拟机从一台主机迁移到另一台主机，过程中业务不中断、用户无感知。

过程：

  1. 将虚拟机配置信息从源主机传送到目的主机上
  2. 传递虚拟机内存（迭代迁移），将虚拟机的初始内存和变更分片同步到目的主机上。
  3. 在源主机上暂停虚拟机并将最后的变更内存传送到目的主机上。
  4. 在目的主机上恢复虚拟机，并在源主机上暂停并清理虚拟机。

4. 虚拟机热迁移的应用场景有哪些？

  1. 计划性停机：当CNA主机需要计划性停机进行更新或者是维护的时候，可以通过虚拟机热迁移将虚拟机
    迁移至其他主机上以保证业务的正常。

  2. 绿色节能：当集群中主机平均负载量较小的时候，通过热迁移将虚拟机集中到部分主机上，并对其余空闲主机作下电操作，起到绿色节能的作用。

  3. 负载均衡：当集群中部分主机负载较高的时候，通过热迁移将虚拟机迁移至较为空闲的主机上，起到负载均衡的作用，提高资源利用效率。

5. 虚拟机热迁移的限制条件由哪些？

虚拟机侧：

  • 虚拟机状态为“运行中”
  • 虚拟机安装了Tools
  • 虚拟机未绑定任何设备（网卡、GPU、USB设备）
  • 虚拟机未与主机绑定

主机侧：

  • 目的主机能与源主机的CPU功能集兼容
  • 目的主机有充足的CPU和内存资源
  • 目的主机状态不能处于“维护模式”
  • 迁移过程中不能将虚拟机下电或重启

网络侧：

  • 源主机和目的主机网络必须互通
  • 目的主机必须关联虚拟机所属端口组所在的DVS（保证业务连续性，迁移之后的IP地址与MAC不变）

存储侧：

  • 数据存储类型为共享存储（SAN、NAS、FusionStorage）

6. 说一下虚拟机热迁移的流量走向？

流量走向：未配置“虚拟机热迁移流量”业务管理接口“时，流量默认通过管理接口，配置勾选“虚拟机热迁移流量”之后，流量由”业务管理接口“承载。

业务管理接口可用服务（6.3）：

  • 虚拟机热迁移

7. 虚拟机热迁移失败的原因有哪些？

  1. 源主机与目的主机网络不互通
  2. 目的主机无法访问虚拟机磁盘
  3. 目的主机CPU、内存资源不足
  4. 源主机与目的主机CPU类型不兼容
  5. 目的主机故障或处于维护状态

8. 介绍一下存储热迁移的定义及迁移过程

存储热迁移：将“运行中“的虚拟机的虚拟磁盘从一个数据存储迁移到另一个数据存储，过程中业务不中断（用户无感知）。

迁移过程：

  1. 在目的数据存储上创建一个空白磁盘。
  2. 将目的磁盘设置为源磁盘的Mirror，业务IO双写至源、目的磁盘中，保证脏数据的实时同步。
  3. 通过迭代技术将源磁盘文件的数据迁移至目的磁盘中。
  4. 短暂的暂停I/O请求，将虚拟机的存储文件从源镜像切换到目的镜像上，完成存储热迁移。

9. 存储热迁移的应用场景有哪些？

应用场景：

  1. 调整服务器间数据存储的负荷
  2. 迁移后对数据存储进行扩容、减容操作

10. 存储热迁移的约束条件有哪些？

约束条件：

  1. 不支持迁移已挂载的”共享“类型的磁盘和链接克隆虚拟机的磁盘
  2. 不支持非持久化磁盘的迁移（条件允许的话可以先关机再迁移）
  3. 不支持跨FusionStorage Block存储资源之间进行迁移。

三、两VM不通

1. 两虚拟机不通，什么原因？

虚拟机内部：

  1. 虚拟机是否蓝屏、死机。

  2. 虚拟机的IP地址、子网掩码、网关是否配置正确/正常获取。

  3. 虚拟机的防火墙是不是限制了ICMP流量。

虚拟机外部：

相同主机，相同网段（内部二层）：

  1. 端口组配置错误（端口组类型、端口组所属的VLAN是否一致）。
  2. 虚拟交换机进程故障。

相同主机，不同网段（三层网络）：

  1. 主机物理网卡故障，导致主机网口与物理交换机的链路中断。
  2. 物理交换机未放行相应的VLAN或物理交换机接错网口。
  3. 物理交换机故障。
  4. 虚拟机流量无法到达网关，或者到达网关后无法路由。

不同主机，相同DVS：

  1. 主机/物理交换机故障。
  2. 虚拟交换机进程故障。
  3. 物理交换机未放行相应VLAN。

2. 如何快速排错？

  1. 先检查两通信虚拟机自身的配置是否正确（IP、网关、子网掩码、arp等）。
  2. 然后通过ping命令，逐层ping定位流量到哪里中断，然后对相应的位置进行检查。
  3. 如果还是无法通信，检查物理交换机是否放行了相应的VLAN。
  4. 最后排查主机、物理交换机硬件是否故障，若故障及时进行更换。

四、常见追问

1. FusionCompute如何搭建？

前期准备：

  • 一台安装用的PC。

  • 到华为官网下载好FusionCompute的CNA、VRM镜像。

安装CNA（两种方式）：

  • 通过挂载ISO，安装单台CNA主机。
    • 在PC机通过BMC IP登录到管理界面。
    • 打开远程虚拟控制台，为主机配置Raid。
    • 选择ISO，重启主机进行安装配置。
  • 通过华为提供的PXE安装工具，批量部署多台CNA主机。
    • 打开PXE工具，配置DHCP、TFTP服务。
    • 设置DHCP地址池范围。
    • 将待安装主机的PXE启动打开。
    • 主机通过DHCP获取地址，然后通过TFTP获取安装镜像。
    • 通过工具对多台主机进行批量部署。

安装VRM（两种方式）：

  • 单节点部署

    • 配置VRM的规格，主机名，IP，子网掩码，网关等。
    • 选择要安装在哪个主机上，配置数据存储，配置权限模式。
    • 开始安装VRM。

  • 主备部署

    • 配置VRM的规格，主机名，浮动IP，子网掩码，网关，仲裁IP等。

    • 然后选择VRM安装在哪个主机上，配置数据存储，配置权限模式（普通模式，三员分立模式）。

    • 开始安装VRM。

2. 磁盘模式有哪些？

从属：快照中包含该从属磁盘。

独立-持久：更改将立即写入磁盘，并且持久磁盘不受快照的影响。

独立-非持久：关闭电源或者恢复快照后，丢失对磁盘的更改。

3. 磁盘的配置模式有哪些？

精简：指定磁盘文件增长的最大空间，空间内数据不置零，以后按需置零。

普通：立即为磁盘文件分配指定大小的空间，并置零。

普通延时置零：立即为磁盘文件分配指定大小的空间，空间暂时不置零，以后按需置零。

4. FusionCompute中有哪些快照？

普通快照：直接对虚拟机进行快照，也叫磁盘快照。

内存快照：先对虚拟机磁盘打快照，再将虚拟机当前的内存状态以及数据保存到快照中。

一致性快照：快照创建时会将虚拟机未保存的缓存数据先保存，再创建快照。

5. HA如何检测到虚拟机故障了？

虚拟机OS故障检测机制：通过虚拟机与主机之间的心跳检测，再配合虚拟机的网络、磁盘IO行为来判断虚拟机是否故障。

HA的触发机制：

  • 当VRM与CNA的心跳中断会触发HA。
  • 当一个虚拟机运行状态消失也会触发HA。

6. HA的特征实现过程？

HA过程：

  1. 当VM或者CNA故障。
  2. VRM查询VM的状态，看是否故障。
  3. 若故障，VRM会去查看VM是否有HA属性。
  4. 若有HA，VRM会将VM信息（规格、卷）保存并发送到集群中可用的CNA上。
  5. 在该CNA上根据VM信息重新拉起一台虚拟机并将原卷及用户卷重新挂载。

7. VRM虚拟机能进行迁移吗？

迁移VRM虚拟机：当VRM虚拟机所在的主机需要进行停机维护的时候，要将VRM虚拟机迁移到其他主机上。

满足条件：

  1. 目标主机上不存在VRM虚拟机。
  2. 目标主机和VRM虚拟机所在主机的CPU型号相同。
  3. 目标主机的虚拟化本地硬盘容量大于140GB。
  4. 已获取VRM主节点和目标主机的”gandalf”和”root”密码。
  5. VRM虚拟机运行正常（主备部署时，确保主备VRM虚拟机均运行正常）。

8. 虚拟机能否实现跨集群迁移？有什么限制条件？

限制条件：

  1. 两集群的内存复用开关一致。
  2. 两集群使用相同的DVS。
  3. 两集群使用相同的共享存储。
